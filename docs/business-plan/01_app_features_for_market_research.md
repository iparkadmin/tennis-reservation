# 予約管理アプリ 特徴まとめ（市場調査用）

> 作成日：2026年3月
> 目的：事業化検討のための市場調査に向けた、アプリ特徴の整理

---

## 1. アプリの本質

**「施設の時間枠を予約し、利用実績を管理する」SaaS**

テニスコート専用に開発されたが、コア機能は**時間枠ベースの予約 + 管理者による利用実績管理**であり、予約対象を変えるだけで汎用化できる構造。

---

## 2. コア機能（汎用化可能）

| 機能 | 内容 |
|------|------|
| **時間枠予約** | 日付×時間枠×施設の3軸で空き管理、重複防止 |
| **利用者登録** | 予約者とは別に当日参加者を登録（同伴者管理） |
| **予約制限** | 日数上限・週上限・予約変更/キャンセル期限を柔軟に設定可能 |
| **管理者ダッシュボード** | ユーザー管理、予約管理、施設管理、カレンダー表示 |
| **利用実績記録** | 利用有無（利用済/無断キャンセル/未記録）、マナー状況、メモ |
| **利用実績集計** | 月単位でユーザー別の問題行動を集計・見える化 |
| **ユーザーブロック** | 問題ユーザーの予約操作を制限 |
| **監査ログ** | 全操作の記録・追跡 |
| **管理者メモ** | ユーザーごとの対応メモ |
| **メール認証** | 登録・パスワードリセット・メール変更 |

---

## 3. 会員向け機能（ユーザー側）

### 認証・プロフィール
- ログイン / 新規登録（メール＋パスワード）
- パスワードリセット（メールリンク）
- メールアドレス変更
- アカウント削除

### 予約
- 予約カレンダー（週ごとの空き状況、施設選択）
- 予約条件（曜日制限、時間枠、日数上限、週上限、重複防止）
- 利用者登録（当日参加者の氏名登録）
- 予約変更・キャンセル（期限制御あり）

### マイページ
- プロフィール編集（氏名・氏名カナ・メール）
- 予約履歴一覧（フィルタ機能付き）
- 予約詳細（変更・キャンセル・利用者編集）

---

## 4. 管理者向け機能

| 機能 | 内容 |
|------|------|
| **ダッシュボード** | 登録ユーザー数、今後の予約数、直近予約一覧 |
| **ユーザー管理** | 検索、ブロック/解除、管理者権限付与、代理予約 |
| **予約管理** | 一覧（フィルタ付き）、詳細、キャンセル |
| **予約カレンダー** | 全予約の視覚的確認（FullCalendar） |
| **施設管理** | 表示名編集、使用可否切り替え |
| **利用実績** | 月単位の集計レポート（無断キャンセル、マナー違反等） |
| **監査ログ** | 操作・テーブルでフィルタ可能な全操作ログ |
| **管理者メモ** | ユーザーごとの対応メモ |

## 5. 技術的強み

| 項目 | 内容 |
|------|------|
| **低コスト運用** | Supabase Free + Vercel Free で月額 ¥0〜数千円 |
| **セキュリティ** | RLS（行レベルセキュリティ）で個人情報を厳密に保護 |
| **レスポンシブ** | PC/スマホ対応 |
| **決済不要** | 施設利用料が無料の場合、そのまま使える |
| **マルチテナント対応** | Supabase のテナント分離で複数施設に対応可能 |

### 技術スタック

| カテゴリ | 技術 |
|---------|------|
| フロントエンド | Next.js 15 (App Router)、React 19、TypeScript |
| スタイリング | Tailwind CSS、Material Design 3 系カラー |
| DB・認証 | Supabase (PostgreSQL + Auth) |
| ホスティング | Vercel |
| カレンダー | FullCalendar |

## 6. 汎用化のための変更点（見積もり）

| 項目 | 現状（テニス専用） | 汎用化後 |
|------|-------------------|----------|
| 施設名 | 「コート」固定 | 設定可能（コート/部屋/スペース等） |
| 時間枠 | 2時間×4枠 固定 | 施設ごとに設定可能 |
| 曜日制限 | 土日祝のみ | 施設ごとに設定可能 |
| 利用ガイド | 湘南アイパーク固定 | テナントごとに設定 |
| 祝日 | 日本の祝日固定 | ロケール対応 |
| 決済 | なし | Stripe 連携（オプション） |

## 7. 適用可能な市場（予約対象の候補）

### 適合条件

1. **時間枠単位で予約する施設**
2. **利用者の管理・実績把握が必要**
3. **無料または月額制（都度決済不要）の施設**
4. **管理者が利用状況を把握したい**

### 日本市場の施設カテゴリ

| カテゴリ | 具体例 | 推定施設数 |
|----------|--------|-----------|
| **テニスコート** | 公営・企業・マンション付帯 | 全国約3万面（公営約1.5万施設） |
| **フットサルコート** | 民間・公営 | 全国約2,000施設 |
| **体育館・アリーナ** | 市区町村の公共体育館 | 全国約5,000施設 |
| **会議室・研修室** | 公共施設・コワーキング | 数万室規模 |
| **BBQ場・キャンプサイト** | 公園・河川敷・民間 | 全国数千カ所 |
| **音楽スタジオ・練習室** | 楽器練習・バンド練習 | 全国数千施設 |
| **多目的グラウンド** | 野球場・サッカー場 | 公営だけで数千施設 |
| **マンション共用施設** | ゲストルーム・パーティールーム・ジム | 大規模マンション数万棟 |
| **社内施設** | 会議室・社用車・備品 | 企業数百万社 |
| **レンタルスペース** | 撮影スタジオ・キッチン | 急成長市場 |

## 8. 差別化要素（既存サービスとの比較ポイント）

| 要素 | 本アプリ | 一般的な予約SaaS |
|------|---------|-----------------|
| 利用実績記録 | あり（利用有無・マナー・メモ） | なし or 簡易 |
| 利用実績集計 | あり（月単位レポート） | なし |
| 同伴者管理 | あり（利用者登録） | なし or 有料オプション |
| ユーザーブロック | あり | なし or 手動対応 |
| 監査ログ | あり | なし or 有料 |
| 管理者メモ | あり | なし |
| 初期・運用コスト | 極めて低い | 月額数万円〜 |
