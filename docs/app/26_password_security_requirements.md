# パスワードセキュリティ要件

乗っ取り防止のため、パスワード要件を強化しています。

## クライアント側の要件

| 項目 | 要件 |
|------|------|
| 最小文字数 | 8文字以上 |
| 最大文字数 | 128文字以内 |
| 大文字 | 1文字以上含む |
| 小文字 | 1文字以上含む |
| 数字 | 1文字以上含む |
| 記号 | 1文字以上含む（例: !@#$% など） |

**表示文言**: 「8文字以上、大文字・小文字・数字・記号をそれぞれ1文字以上含めてください」

## 適用箇所

- 新規登録（AuthForm）
- パスワードリセット後の新パスワード設定（ログインページ）

## Supabase ダッシュボードの設定（推奨）

クライアント側のバリデーションに加え、Supabase 側でも最小文字数を設定することを推奨します。

1. **Supabase ダッシュボード** → プロジェクトを選択
2. **Authentication** → **Settings**（または **Providers** の Email）
3. **Minimum Password Length** を **8** に設定
4. 保存

これにより、クライアント側のバリデーションをバイパスした場合でも、Supabase Auth が8文字未満のパスワードを拒否します。
