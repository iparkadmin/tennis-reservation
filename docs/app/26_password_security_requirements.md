# パスワードセキュリティ要件

乗っ取り防止のため、パスワード要件を強化しています。**Supabase Auth の設定に合わせて**クライアント側のバリデーションと UI を実装しています。

## 要件（Supabase と一致）

| 項目 | 要件 |
|------|------|
| 最小文字数 | 8文字以上 |
| 最大文字数 | 128文字以内 |
| 小文字 | 1文字以上含む（a-z） |
| 大文字 | 1文字以上含む（A-Z） |
| 数字 | 1文字以上含む（0-9） |
| 記号 | 1文字以上含む（!@#$% など） |

**表示文言**: 「8文字以上、小文字・大文字・数字・記号をそれぞれ1文字以上含めてください」

## 適用箇所

- 新規登録（AuthForm）: チェックリスト付き
- パスワードリセット後の新パスワード設定（ログインページ）: チェックリスト付き

## Supabase ダッシュボードの設定

1. **Supabase ダッシュボード** → プロジェクトを選択
2. **Authentication** → **Providers** → **Email** をクリック
3. 以下を設定:
   - **Minimum password length**: 8
   - **Password Requirements**: "Lowercase, uppercase letters, digits and symbols (recommended)"
4. **Save** をクリック

これにより、クライアント側のバリデーションをバイパスした場合でも、Supabase Auth が弱いパスワードを拒否します。
