# マイページ機能 要件定義

## 📋 概要

マイページ機能として、以下を実装します：
1. プロフィール確認・編集
2. 予約状況の表示
3. 予約の変更・キャンセル機能

---

## 1. ページ構成

### 1.1 マイページ（`/mypage`）

**機能：**
- プロフィール情報の確認（氏名、メールアドレス）
- 予約状況サマリー（今月の予約数、次の予約日時）
- 最近の予約（直近3件）
- クイックアクション（予約カレンダー、予約履歴、プロフィール編集へのリンク）

**レイアウト：**
```
┌─────────────────────────────────┐
│ マイページ                        │
├─────────────────────────────────┤
│ プロフィール情報                  │
│ - 氏名: 村松 龍仁                │
│ - メール: xxx@example.com        │
│ [プロフィール編集]                │
├─────────────────────────────────┤
│ 予約状況サマリー                  │
│ - 今月の予約数: 3件              │
│ - 次の予約: 2026年1月16日 11:00 │
├─────────────────────────────────┤
│ 最近の予約                        │
│ - 2026年1月16日 11:00-11:30     │
│   [詳細] [変更] [キャンセル]      │
│ - 2026年1月23日 14:00-15:00     │
│   [詳細] [変更] [キャンセル]      │
└─────────────────────────────────┘
```

### 1.2 プロフィール編集（`/member/profile`）

**機能：**
- プロフィール情報の表示・編集
- 氏名、氏名（カナ）、メールアドレスの表示（メールは編集不可）
- 保存機能

**入力項目：**
- 氏名（必須）
- 氏名（カナ）（必須）
- メールアドレス（必須、変更不可または変更可能）
### 1.3 予約履歴・詳細（`/member/reservations`）

**機能：**
- 予約一覧の表示（過去・未来）
- 予約詳細の表示
- 予約の変更（日時変更）
- 予約のキャンセル

**フィルター：**
- 今月
- 来月
- 過去
- すべて

### 1.4 予約変更（`/member/reservations/[id]`）

**機能：**
- 既存予約の日時変更
- 変更可能期間のバリデーション（前日まで）
- 変更後の確認画面

## 2. データベース設計（更新）

### 2.1 `profiles`テーブルの拡張

現在のテーブルに以下を追加：

```sql
ALTER TABLE profiles 
ADD COLUMN IF NOT EXISTS full_name_kana TEXT;
```
※ 電話番号（phone）は仕様変更により廃止（マイグレーション 18 で削除）。

### 2.2 `reservations`テーブルの拡張

予約変更機能のために以下を追加：

```sql
ALTER TABLE reservations 
ADD COLUMN IF NOT EXISTS contact_notes TEXT,
ADD COLUMN IF NOT EXISTS reservation_number TEXT UNIQUE;
```

予約番号は自動生成（例：`7w3Lf8iXQ`）

## 3. 機能詳細

### 3.1 プロフィール確認・編集

**表示項目：**
- 氏名
- 氏名（カナ）
- メールアドレス
- 登録日時

**編集機能：**
- 氏名、氏名（カナ）を編集可能
- メールアドレスは変更不可（Supabase Authで管理）

### 3.2 予約状況の表示

**サマリー情報：**
- 今月の予約数
- 来月の予約数
- 次の予約日時
- 過去の予約数

**最近の予約：**
- 直近3件を表示
- 予約日時、時間帯
- 詳細・変更・キャンセルボタン

### 3.3 予約の変更

**変更可能条件：**
- 予約日の前日まで変更可能
- 当日以降は変更不可（キャンセルのみ可能）

**変更フロー：**
1. 予約詳細ページで「変更」ボタンをクリック
2. カレンダーで新しい日時を選択
3. 変更確認画面
4. 変更実行

**制約：**
- 1日最大2時間制限を遵守
- 重複予約の防止
- 営業時間内（9:00-17:00）の予約のみ

### 3.4 予約のキャンセル

**キャンセル可能条件：**
- 予約日の前日までキャンセル可能
- 当日以降はキャンセル不可

**キャンセルフロー：**
1. 予約一覧または詳細で「キャンセル」ボタンをクリック
2. 確認ダイアログ
3. キャンセル実行

## 4. UI/UX要件

### 4.1 マイページ

- **カード型レイアウト** - 各セクションをカードで表示
- **クイックアクション** - よく使う機能へのショートカット
- **レスポンシブ対応** - スマートフォンでも見やすい

### 4.2 プロフィール編集

- **フォーム形式** - 入力フィールドと保存ボタン
- **バリデーション** - 必須項目のチェック
- **保存確認** - 保存成功時のメッセージ表示

### 4.3 予約変更

- **カレンダー表示** - 新しい日時を選択
- **変更前後の比較** - 変更前と変更後の日時を表示
- **確認画面** - 変更内容の最終確認
