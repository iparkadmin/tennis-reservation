# 管理画面 使用ガイド

## アクセス方法

1. **管理者アカウントでログイン**（`profiles.role = 'admin'` のユーザー）
2. ブラウザで **`/admin`** にアクセス  
   - 例: `https://tennis-reservation-one.vercel.app/admin`
3. 一般ユーザーがアクセスするとトップページへリダイレクトされます

---

## メニュー一覧

| メニュー | パス | 機能 |
|----------|------|------|
| ダッシュボード | `/admin` | ユーザー数・予約数のサマリー、直近の予約一覧 |
| ユーザー一覧 | `/admin/users` | 全ユーザーの検索・一覧、ブロック状態の確認 |
| 予約一覧 | `/admin/reservations` | 日付・コートでフィルター、予約の検索 |
| 予約カレンダー | `/admin/calendar` | 週単位で全予約を俯瞰 |
| コート管理 | `/admin/courts` | コートの表示名・使用可否の編集 |
| 監査ログ | `/admin/audit-logs` | 予約・利用者の操作履歴 |

---

## 各機能の使い方

### ダッシュボード（/admin）

- 登録ユーザー数・予約数を確認
- 直近の予約一覧を表示
- 「詳細」リンクから予約詳細へ移動

### ユーザー一覧（/admin/users）

- **検索**: 氏名・カナ・メールで絞り込み
- **状態列**: ブロック中のユーザーは「ブロック中」と表示
- **詳細**: ユーザー行の「詳細」をクリック

### ユーザー詳細（/admin/users/[id]）

| セクション | 操作 |
|------------|------|
| **プロフィール** | 氏名・メール・登録日を表示。**ブロック/ブロック解除**ボタンでユーザーをブロック |
| **登録利用者** | 家族・友人などの利用者を追加・編集・削除 |
| **運営メモ** | 個別対応の記録を追加（管理者のみ閲覧可能） |
| **今後の予約** | 「予約を代行作成」でユーザーに代わって予約を作成 |
| **過去の予約** | 予約履歴を表示、詳細リンクから予約詳細へ |

**ブロック機能**: ブロックされたユーザーは予約の作成・変更・キャンセルができません。管理者はブロックユーザーへの予約代行は可能です。

### 予約一覧（/admin/reservations）

- **フィルター**: 開始日・終了日・コートで絞り込み
- 予約者名・予約番号・日時を表示
- 「詳細」から予約詳細へ

### 予約詳細（/admin/reservations/[id]）

- 予約内容の確認
- **代理キャンセル**: ユーザーに代わって予約をキャンセル
- 「会員画面で変更」: ユーザー向けの予約変更ページへリンク

### 予約カレンダー（/admin/calendar）

- コートを選択して週単位で表示
- 「← 前週」「次週 →」で週を移動
- 予約済みスロットに予約者名を表示、クリックで予約詳細へ
- 予約可能期間（今日から1か月以内）を超える週は「次週」が無効

### コート管理（/admin/courts）

- **表示名**: カレンダー等に表示する名前を編集
- **使用可否**: 無効にしたコートは予約カレンダーに表示されない

### 監査ログ（/admin/audit-logs）

- 操作・テーブルでフィルター
- 直近200件を表示
- 予約・利用者の作成・更新・削除履歴を確認

## 管理者の追加方法

Supabase SQL Editor で実行:

```sql
UPDATE profiles SET role = 'admin' WHERE email = '管理者のメールアドレス@example.com';
```

## トラブルシューティング

| 現象 | 確認・対処 |
|------|------------|
| 管理画面にアクセスできない（トップへ飛ぶ） | `profiles.role = 'admin'` か確認。ログアウトして再ログイン |
| データが表示されない | ブラウザコンソールのエラー確認。Supabase RLS が正しく設定されているか確認 |
| ブロックボタンが表示されない | 自分自身のユーザー詳細ではブロックボタンは表示されない（自己ブロック防止） |
