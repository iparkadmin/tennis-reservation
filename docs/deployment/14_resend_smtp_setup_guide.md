# Resend SMTP設定ガイド

> **目的：** Resend.comを使用してSupabase Authのメールをカスタムドメイン（iPark Institute Co., Ltd.）から送信する

---

## 📧 Resendの特徴

- **無料枠：** 月3,000通まで（1日100通まで）
- **有料プラン：** 月$20から（月50,000通）
- **特徴：**
  - 設定が非常に簡単
  - モダンなAPI設計
  - ドキュメントが充実
  - React Emailとの統合が可能
  - **DNS設定なしでも使用可能**（メールアドレス認証のみ）
  - 高い配信率
  - 詳細な送信ログ

---

## 🔧 設定手順

### Step 1: Resendアカウントの作成

#### 1.1 アカウント登録

1. https://resend.com にアクセス
2. **Sign Up** をクリック
3. 以下の情報を入力：
   - **Email:** メールアドレス
   - **Password:** パスワード
4. **Create Account** をクリック
5. メール認証を完了

#### 1.2 ダッシュボードにログイン

1. https://resend.com/login にアクセス
2. 登録したメールアドレスとパスワードでログイン

---

### Step 2: API Keyの作成

#### 2.1 API Key設定画面を開く

1. Resendダッシュボードにログイン
2. 左メニューから **API Keys** を選択
3. **Create API Key** をクリック

#### 2.2 API Keyを作成

1. 以下の情報を入力：
   - **Name:** `Supabase SMTP`（任意の名前）
   - **Permission:** **Sending access** を選択
2. **Add** をクリック
3. **API Keyをコピー**（この画面でしか表示されないため、必ずコピー）

**重要：** API Keyはこの画面でしか表示されないため、必ずコピーして安全な場所に保存してください。

---

### Step 3: 送信元メールアドレスの認証（簡単な方法）

#### 3.1 メールアドレス認証（DNS設定不要）

1. 左メニューから **Domains** を選択
2. **Add Domain** をクリック
3. ドメイン名を入力（例：`ipark-institute.com`）
4. **Add** をクリック

**または、より簡単な方法：**

1. 左メニューから **Emails** → **Verified Emails** を選択
2. **Add Email** をクリック
3. 送信元メールアドレス（`noreply@ipark-institute.com`）を入力
4. **Add** をクリック
5. 認証メールが届くので、メール内のリンクをクリックして認証

**注意：** メールアドレス認証のみでも使用可能です。DNS設定は不要です。

---

### Step 4: SupabaseでカスタムSMTPを設定

#### 4.1 Supabaseダッシュボードにアクセス

1. https://supabase.com/dashboard にログイン
2. プロジェクトを選択（tennis court reservation）
3. 左メニューから **Settings** → **Auth** を選択

#### 4.2 SMTP設定を開く

1. **Auth** 設定画面をスクロール
2. **SMTP Settings** セクションを探す
3. **Enable Custom SMTP** のトグルを **ON** に設定

#### 4.3 SMTP情報を入力

以下の情報を入力します：

| 項目 | 値 |
|------|-----|
| **Host** | `smtp.resend.com` |
| **Port** | `587`（STARTTLS推奨）または `465`（SSL） |
| **Username** | `resend`（固定） |
| **Password** | Step 2.2で作成したAPI Key |
| **Sender email** | `noreply@ipark-institute.com`（Step 3で認証済みのメールアドレス） |
| **Sender name** | `iPark Institute Co., Ltd.` |

**注意：** 
- Usernameは常に `resend`（固定）です
- PasswordにはAPI Keyを使用します

#### 4.4 設定を保存

1. すべての情報を入力したら、**Save** をクリック
2. SupabaseがSMTP接続をテストします
3. 成功メッセージが表示されれば設定完了

**エラーが表示された場合：**
- Host名が正しいか確認（`smtp.resend.com`）
- Port番号を確認（`587` または `465`）
- Usernameが `resend` になっているか確認
- PasswordにAPI Keyが正しく入力されているか確認
- 送信元メールアドレスが認証済みか確認

## ✅ 設定確認

### テスト方法

1. **新規登録時のメール認証をテスト**
   - テスト用のメールアドレスで新規登録を実行
   - メールが届くことを確認
   - 送信元が `iPark Institute Co., Ltd. <noreply@ipark-institute.com>` になっているか確認

2. **パスワードリセットメールをテスト**
   - ログイン画面で「パスワードを忘れた場合」をクリック
   - テスト用のメールアドレスを入力
   - メールが届くことを確認
   - 送信元を確認

### Resendダッシュボードで送信ログを確認

1. Resendダッシュボードにログイン
2. 左メニューから **Emails** → **Logs** を選択
3. 送信されたメールの一覧が表示されます
4. 各メールの送信状況（成功、失敗、バウンス等）を確認できます

## 🔍 トラブルシューティング

### SMTP接続エラー

**エラー：** `SMTP connection failed`

**対処法：**
1. **Host名を確認**
   - 正しいHost名：`smtp.resend.com`
   - タイポがないか確認

2. **Port番号を確認**
   - STARTTLS: `587`
   - SSL: `465`
   - 両方を試してみる

3. **認証情報を確認**
   - Usernameが `resend`（固定）になっているか確認
   - PasswordにAPI Keyが正しく入力されているか確認
   - ResendダッシュボードでAPI Keyが正しく作成されているか確認

4. **送信元メールアドレスの認証を確認**
   - `noreply@ipark-institute.com` が認証済みか確認
   - Resendダッシュボードの **Emails** → **Verified Emails** で確認

### メールが届かない

**対処法：**
1. **スパムフォルダを確認**
   - メールがスパムフォルダに振り分けられていないか確認

2. **送信ログを確認**
   - Resendダッシュボードの **Emails** → **Logs** で送信ログを確認
   - エラーが発生していないか確認
   - バウンスやブロックされていないか確認

3. **レート制限を確認**
   - 無料枠の制限（1日100通、月3,000通）に達していないか確認
   - Resendダッシュボードの **Settings** → **Usage** で送信数を確認

4. **送信元メールアドレスの認証を確認**
   - 送信元メールアドレスが認証済みか確認
   - 認証が完了していない場合は、認証メール内のリンクをクリック

### 送信元が正しく表示されない

**対処法：**
1. **Sender emailとSender nameを確認**
   - SupabaseのSMTP設定で正しく入力されているか確認
   - 送信元メールアドレスがResendで認証されているか確認

2. **メールクライアントの表示を確認**
   - メールクライアントによって表示が異なる場合があります
   - 複数のメールクライアント（Gmail、Outlook等）で確認

## 📊 Resendの無料枠について

### 無料枠の制限

- **送信数：** 月3,000通まで
- **1日の制限：** 100通まで
- **レート制限：** 2リクエスト/秒
- **品質基準：**
  - バウンス率：4%以下
  - スパム率：0.08%以下

### 無料枠で十分な場合

以下のような用途であれば、無料枠で十分です：

- 新規登録時のメール認証（月100-200通程度）
- パスワードリセットメール（月50-100通程度）
- 予約完了確認メール（月200-500通程度）
- 合計：月350-800通程度

**注意：** 予約前日リマインドメール等の追加メールを実装する場合は、送信数が増えるため、有料プランへのアップグレードを検討してください。

### 有料プランへのアップグレード

月3,000通を超える場合は、以下のプランがあります：

- **Pro:** 月$20（月50,000通）
- **Business:** 月$80（月200,000通）
- **Enterprise:** カスタムプラン

## 🎯 Resendを選ぶ理由

### 1. 設定が簡単

- **DNS設定不要：** メールアドレス認証のみで使用可能
- **API Keyのみ：** 複雑なSMTP認証情報は不要
- **わかりやすいUI：** ダッシュボードがシンプルで使いやすい

### 2. 無料枠が実用的

- **月3,000通：** SMTP2GO（月1,000通）の3倍
- **1日100通：** 十分な送信量
- **永続的：** 無料枠が永続的に利用可能

### 3. 開発者向け

- **モダンなAPI：** RESTful APIとSMTPの両方をサポート
- **React Email対応：** React Emailとの統合が容易
- **詳細なドキュメント：** 充実したドキュメント

### 4. 高品質

- **高い配信率：** 優れた配信率を実現
- **詳細なログ：** 送信ログ、開封率、クリック率を追跡
- **Webhook対応：** イベント通知に対応

## 🔐 セキュリティ注意事項

1. **API Keyの管理**
   - API Keyは機密情報として扱う
   - 環境変数やシークレット管理サービスで管理
   - 定期的にローテーション

2. **送信元メールアドレスの認証**
   - 必ず送信元メールアドレスを認証する
   - 未認証のメールアドレスから送信すると、スパムとして扱われる可能性が高い

3. **アカウントのセキュリティ**
   - Resendアカウントに2要素認証（2FA）を有効化（推奨）
   - 定期的にパスワードを変更

## 💡 DNS設定について（オプション）

Resendでは、**DNS設定なしでも使用可能**です。メールアドレス認証のみで送信できます。

ただし、より高い配信率とブランディングを実現するために、ドメイン認証（DNS設定）を行うことも可能です。DNS設定は以下の場合に推奨されます：

- 大量のメールを送信する場合
- より高い配信率が必要な場合
- ブランディングを強化したい場合

DNS設定を行う場合は、Resendダッシュボードの **Domains** セクションで、DNSレコードの設定手順が表示されます。
