# テニスコート予約システム 要件定義書（無料サービス版・改訂）

> **改訂目的：** すべて無料のサービスとして、支払い関連機能を除外し、サイト構造を再整理

---

## 1. ビジネス要件整理（改訂版）

### 1.1 解決する課題（ペイン）

| 課題 | 詳細 |
|------|------|
| **予約管理の煩雑さ** | 電話やメールでの手動管理による、ダブルブッキングや転記ミスの発生 |
| **利用者側の利便性** | 空き状況がリアルタイムで分からず、予約の確定までに時間がかかる |
| **無料サービスの提供** | すべて無料で提供し、支払い情報の入力や管理を不要にする |

### 1.2 ターゲット顧客（ペルソナ）

- **ペルソナ：** 社内福利厚生施設（テニスコート）の利用者
- **利用形態：** 完全無料、会員登録必須（~~ゲスト予約は実装しない~~）

### 1.3 収益モデル

- **完全無料サービス** - 支払い情報は一切不要
- **管理コスト削減** - 予約管理の自動化による間接的な利益（ROI）

### 1.4 競合との差別化

- **完全無料：** 月額プラン、回数券、支払い機能は一切不要
- **シンプルな予約フロー：** 会員登録後すぐに予約可能（~~ゲスト予約は実装しない~~）
- **特定ルール特化：** 「最大2時間」「土日祝限定」といった、汎用ツールでは設定が面倒な特定ルールをプリセット

### 1.5 効果的なアプローチ

- **LP一体型：** サービス説明、利用規約、ログインボタンが1枚で完結するシンプルな構成
- **会員限定サービス：** セキュリティとプライバシー保護のため、会員登録必須
- **自動通知：** 予約完了時・前日リマインドのメール通知（正規版）

```

├─ ホーム（/）

│   └─ メール認証（/auth/verify）
│
├─ 予約フロー（会員）
│   ├─ 予約選択（/dashboard）
│   │   ├─ 日付選択
│   │   └─ 時間選択（9:00-17:00、1時間単位）
│   ├─ 最終確認（/reservation/confirm）

└─ 会員メニュー（/member）
    ├─ ホーム（/member/home）

### 2.2 削除する機能（支払い関連）

以下の機能は**一切実装しない**：

- ❌ 月額プラン
- ❌ 回数券
- ❌ お支払い予定
- ❌ 決済情報入力
- ❌ 支払い履歴
- ❌ クレジットカード登録
- ❌ 請求書・領収書発行

## 3. 機能要件定義（改訂版）

### 3.1 MVP機能（完全無料版）

| 機能カテゴリ | 詳細 |
|--------------|------|
| **認証** | Supabase Auth（メール/パスワード）によるログイン・ログアウト（会員登録必須） |
| **予約カレンダー** | 9:00-17:00（1時間単位）の空き状況表示。土日祝のみアクティブ |
| **予約登録** | 1ユーザー1日最大2時間まで。1時間単位で選択 |
| **予約確認** | 予約日時、会員情報、連絡事項、利用規約同意 |
| **予約完了** | 予約番号発行、確認メール送信 |
| **マイページ** | 自分の現在の予約確認、予約履歴、キャンセル機能 |
| **プロフィール管理** | 会員情報の編集（氏名、メールアドレス） |
| **バリデーション** | 同一時間帯の重複予約禁止。最大時間超過の制限 |
| **利用上の注意表示** | 来館申請の注意をトップ、予約カレンダー上、予約履歴・予約詳細の下部に表示。文言は共通定数で一元管理（7.3 参照） |

### 3.2 正規版への拡張機能

- **n8n自動通知：** 予約完了時・前日リマインドのメール通知
- **複数コート管理：** コートA、コートBの選択機能
- **雨天中止の一斉通知：** 天候による中止時の自動通知
- **管理者ダッシュボード：** 利用率のグラフ化、特定ユーザーの予約制限

6. 予約完了（/reservation/complete）
   ├─ 予約番号表示

   ↓

   └─ 予約履歴に反映
```

## 5. データベース設計（改訂版）

### 5.1 テーブル構造

#### Table: `profiles` (ユーザー情報)

| カラム名 | 型 | 説明 |
|----------|-----|------|
| `id` | uuid | Primary Key, references auth.users |
| `full_name` | text | ユーザー名 |
| `full_name_kana` | text | ユーザー名（カナ） |
| `email` | text | メールアドレス |
| `created_at` | timestamp | 作成日時 |
| `updated_at` | timestamp | 更新日時 |

#### Table: `reservations` (予約データ)

| カラム名 | 型 | 説明 |
|----------|-----|------|
| `id` | uuid | Primary Key |
| `user_id` | uuid | Foreign Key to profiles.id（NOT NULL：会員のみ予約可能） |
| `booking_date` | date | 予約日 |
| `start_time` | time | 開始時間 |
| `end_time` | time | 終了時間 |
| `contact_notes` | text | 連絡事項（任意） |
| `reservation_number` | text | 予約番号（一意） |
| `created_at` | timestamp | 作成日時 |

#### Table: `inquiries` (お問い合わせ)

| カラム名 | 型 | 説明 |
|----------|-----|------|
| `id` | uuid | Primary Key |
| `user_id` | uuid | Foreign Key to profiles.id（NULL可） |
| `subject` | text | 件名 |
| `message` | text | 本文 |
| `created_at` | timestamp | 作成日時 |

## 7. UI/UX要件

### 7.1 支払い関連の表示禁止

- **料金表示：** すべて「¥0」または「無料」と表示
- **支払いボタン：** 一切表示しない
- **決済情報入力：** 一切表示しない
- **請求書・領収書：** 一切表示しない

### 7.2 予約完了画面

- **予約番号：** 必ず表示（例：`7w3Lf8iXQ`）
- **予約日時：** 必ず表示
- **確認メール送信通知：** 必ず表示
- **料金表示：** 表示しない（または「無料」と表示）

### 7.3 来館申請の注意表示

来館申請とご予約者様の参加必須について、以下の画面に共通文言（`VISIT_APPLICATION_NOTICE`）を表示する。

| 画面 | 表示位置 |
|------|----------|
| トップ | ご利用ルールの上（注意用カード） |
| 予約カレンダー（/dashboard） | カレンダー上の説明ブロック内 |
| 予約履歴（/member/reservations） | ページ下部 |
| 予約詳細・変更（/member/reservations/[id]） | ページ下部 |

**文言：** ご利用の際には来館申請が必要となりますので、ご予約者様のお名前で、必ずご利用前日までに当日参加される方の来館申請を完了してください。また、ご予約者様ご自身のご参加を必須とし、ご利用ください。
